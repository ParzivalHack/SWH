First Steps with SecureWebHost

Beginner

Welcome to SecureWebHost! In this first tutorial, you'll learn how to install and set up SecureWebHost, then launch your first secure web server in just 30 seconds.

üñ•Ô∏è System Requirements

üìã Minimum Requirements

  • Operating System: Windows 10+, macOS 10.14+, or Linux (Ubuntu 18.04+)
  • Python: Python 3.8 or higher
  • RAM: 512MB minimum (2GB recommended)
  • Storage: 50MB free space
  • Network: Internet connection for initial setup

üì¶ Installation Process

1
Clone Repository

First, clone the SecureWebHost repository:

git clone https://github.com/ParzivalHack/SWH.git cd SWH
2
Install Dependencies

Install all required Python packages:

pip install -r requirements.txt
3
Launch with GUI

Start SecureWebHost with the beautiful GUI:

python swh.py --gui
4
Select Website Folder

Click "Browse Folder" and select your website directory, then "Include All".

üéâ Congratulations!

You've successfully installed SecureWebHost! Your website is now running with enterprise-grade security protection.

üß™ Testing Your Setup

Open your browser and go to http://localhost:8080 to see your website. Try this test URL to verify WAF protection:

http://localhost:8080/?test=<script>alert('xss')</script>

This should be blocked by the WAF and appear in your Security Dashboard.

Deploying to GitHub Pages

Intermediate

Learn how to deploy your website to GitHub Pages with one click using real API integration.

üìñ GitHub Pages Overview

GitHub Pages is a free static site hosting service that's perfect for SecureWebHost projects:

  • Free Hosting: No cost for public repositories
  • HTTPS by Default: Secure connections automatically
  • Custom Domains: Use your own domain name
  • Global CDN: Fast loading worldwide
  • Version Control: Every deployment is tracked

üîê GitHub Authentication Setup

1
Create Access Token
  1. Go to GitHub.com ‚Üí Settings
  2. Developer settings ‚Üí Personal access tokens
  3. Generate new token (classic)
  4. Select scopes: repo, read:user, user:email
2
Add to SecureWebHost
  1. Go to Deployment ‚Üí GitHub Pages
  2. Click "Configure GitHub Integration"
  3. Paste your Personal Access Token
  4. Enter your GitHub username
  5. Click "Verify Connection"

üîê Security Important

Copy your token immediately after generation - GitHub will never show it again! Store it securely.

üöÄ One-Click Deployment

Once authenticated, deployment is incredibly simple:

Repository Name: my-awesome-website Description: My website built with SecureWebHost Visibility: Public (required for free GitHub Pages) Branch: gh-pages (automatically created)

Click "Deploy to GitHub Pages" and watch the magic happen:

‚òÅÔ∏è Cloudflare Tunnel Overview

Cloudflare Tunnel provides secure, encrypted connections between your server and Cloudflare's edge, eliminating the need for port forwarding:

  • No Port Forwarding: Works behind firewalls and NAT
  • Automatic SSL: Free SSL certificates for all domains
  • DDoS Protection: Enterprise-grade protection included
  • Global CDN: Your site loads fast worldwide
  • Zero Trust: Built-in security and access controls

üîß Cloudflare Setup

1
Domain Setup
  1. Add your domain to Cloudflare
  2. Update nameservers at your registrar
  3. Wait for DNS propagation (5-30 minutes)
2
Tunnel Configuration

In SecureWebHost's Cloudflare Tunnel tab:

  1. Enter your domain name
  2. Configure tunnel settings
  3. Click "Start Production Tunnel"

Custom Honeypot Creation

Advanced

Create custom honeypots to catch attackers and analyze their behavior for enhanced security intelligence.

üçØ Honeypot Overview

Honeypots are decoy targets designed to attract and trap attackers. SecureWebHost includes 9 pre-configured honeypots and the ability to create custom ones:

Pre-configured Honeypots:

  • /admin - Generic admin panel
  • /wp-admin - WordPress admin
  • /phpmyadmin - Database admin
  • /.env - Environment files
  • /backup - Backup directories
  • /config - Configuration files
  • /api - API endpoints
  • /test - Testing directories
  • Hidden form fields - Invisible traps

üéØ Creating Custom Honeypots

1
Honeypot Design

Navigate to: Security Center ‚Üí Honeypot Management ‚Üí Create Custom

Design honeypots specific to your application's threat model.

2
Path-Based Honeypots

Create fake endpoints that shouldn't exist in your application.

3
Form Field Honeypots

Add invisible form fields that only bots will fill out.

4
File-Based Honeypots

Create decoy files that trigger alerts when accessed.
